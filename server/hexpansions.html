Adding Hexpansion support

Each supported Hexpansion is a small Python class in app/hexpansion/. It declares a name, the commands it can receive, and how to tell when a command has been carried out. Everything else is handled by the base class.

What a module needs

Extend HexpansionModule from app/hexpansion/base.py and define three things:

Attribute / methodWhat to do
FRIENDLY_NAME A string that exactly matches the name in the Hexpansion's EEPROM manifest. The base class uses it to detect that the hardware is plugged in.
COMMAND_OPTIONS A list of every command this Hexpansion can be asked to perform.
on_button_down(event) Called for every button event. When the event satisfies the current command, set self.last_status = CommandStatus.PASSED.

The three modules that ship with the game — Tildagon2024, MegaDrive and GPS, in app/hexpansion/ — are the best reference.

Minimal example

A Hexpansion with a single big red button:

from .base import HexpansionModule, CommandStatus


class BigRedButtonModule(HexpansionModule):
    FRIENDLY_NAME = "BigRedButton"  # must match EEPROM name exactly
    COMMAND_OPTIONS = ["press"]

    def on_button_down(self, event):
        button = getattr(event, "button", None)
        if button is None:
            return
        if button.group != "BigRedButton":
            return
        if self.current_command == "press":
            self.last_status = CommandStatus.PASSED

Because the button press sets self.last_status directly, nothing else needs overriding.

Sensor-based commands (like GPS or shake) can't use button events. Override check_command() instead and read your hardware there — it's called on every poll (~1 s), so keep it fast and non-blocking. See GPS.py for a worked example.

Register it

  1. Save your class as app/hexpansion/MyHexpansion.py.
  2. Add it to MODULE_TYPES in app/hexpansion/__init__.py:
from .MyHexpansion import MyHexpansionModule

MODULE_TYPES = [
    Tildagon2024Module,
    MegaDriveModule,
    GPSModule,
    MyHexpansionModule,  # add here
]
FRIENDLY_NAME must exactly match the EEPROM. The name comes from the Hexpansion's firmware manifest — check the hexpansion-firmwares repo or run scripts/generate_hexpansion_names.py to regenerate the lookup table.

Test it on hardware

Copy the app to the badge and reset it:

pipx run mpremote cp -r app/* :/apps/spaceteam/ && pipx run mpremote reset

Then, with your Hexpansion plugged in, open the Race Condition app and select Test modules from the main menu. The badge steps through every command for each detected Hexpansion; perform the action and it advances automatically (hold Cancel for 2 s to skip). If your module doesn't appear it wasn't detected — double-check that FRIENDLY_NAME matches the EEPROM exactly.
